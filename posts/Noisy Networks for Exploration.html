                                                # Noisy Networks for Exploration

                                                # Basic Information

                                                • 2018 ICLR
                                                • Meire Fortunato, Mohammad Gheshlaghi Azar, Bilal Piot, et al. @ Google Deepmind

                                                # 問題描述

                                                在過去的 RL 當中我們往往仰賴對 agent 的 policy 增加 randomness 去增加 exploration，例如 ϵ-greedy 和 entropy regularization 等。不過這樣的做法往往只能在較於簡單的環境當中有比較有效率的探索，然而在現實狀況下往往並不會如此簡單，而這種探索的困難度甚至是指數性地成長。

                                                例如在 Alpha Go 當中就至少探索了上千億甚至到幾兆個模擬的遊戲狀態

                                                Info

                                                論文中有提及在 Matthieu Geist, Olivier Pietquin (2014)open in new window 有提及一個使用 Neural Network 的方法，不過並沒有保證收斂，因此仍然沒有解決問題。

                                                因此這一篇論文提出一個方法試圖去消除 exploration 效率與品質的問題。

                                                # Parameter Space Noise for Exploration

                                                2017 年由 OpenAI 發表在 ICLR 的 paper。其方法與這一篇可說是大同小異。

                                                在過往的研究可以發現到說往往我們在設計讓 agent 有更多的 exploration 的時候都是透過增加 noise 來達成。

                                                Info

                                                舉例而言，ϵ\epsilonϵ-greedy 就是在 action space 上增加了 noise，讓選擇更多樣，以達成 exploration。

                                                而在 A3C 當中加上 Entropy Regularization，是在 Loss 上鼓勵 policy 的亂度越高越好，達到鼓勵 exploration 的效果。

                                                Image from OpenAI - Better exploration with parameter noiseopen in new window

                                                核心的概念很簡單，過去增加探索的方法大多都是在 action space 上增加 noise，而這裡則選擇在 parameter space 上增加 noise，並且達到了很棒的效果。

                                                Tips

                                                ϵ\epsilonϵ-greedy 就像是獵人裡面的凱特，行動之前需要先看運氣抽接下來使用的武器，即便自己知道當下用哪一個 action 比較好，卻會受到 ϵ\epsilonϵ 的限制。

                                                而在 parameter space 加上 noise 就像是可以換個角度去想其他人會怎麼做，試著用那一個人的做法走過一次，得到不同的經驗。

                                                相較之下，action space 加 noise 就比較像是在亂試，反之在 parameter space 上加 noise 就比較有系統性一些。

                                                具體來說就是他們試圖在 parameter 上加上 Gaussian Noise。

                                                θ~=θ+N(0,σ2I) \tilde{\theta} = \theta + \mathcal{N}(0, \sigma^2I) θ~=θ+N(0,σ2I)

                                                # DQN

                                                透過 Neural Network 去學習 optimal action value function Q∗Q^*Q∗。 Action 的決定上採用了 ϵ\epsilonϵ-greedy。

                                                Image from Ziyu Wang et al. (2015)open in new window

                                                於是他們定義了底下的 Loss function 去試圖得到 Q∗Q^*Q∗。

                                                L(θ)=E(s,a,r,y)∼D[(r+γmax⁡b∈AQ(y,b;θ−)−Q(s,a;θ))2] L(\theta) = \mathbb{E}_{(s, a, r, y) \sim D} \left[ \left( \textcolor{red}{r + \gamma \max_{b \in A} Q(y, b; \theta^-)} - Q(s, a; \theta)\right)^2 \right] L(θ)=E(s,a,r,y)∼D​[(r+γb∈Amax​Q(y,b;θ−)−Q(s,a;θ))2]

                                                • DDD 是上一個 replay buffer 的 transition distribution
                                                  • state sss
                                                  • action aaa
                                                  • reward r=R(s,a)r = R(s, a)r=R(s,a)
                                                  • probability y∼P(⋅∣s,a)y \sim P(\cdot | s, a)y∼P(⋅∣s,a)
                                                • θ−\theta^-θ− 是被固定的參數

                                                Tips

                                                DQN 帶來了幾個好處

                                                • Experience Replay 透過儲存 Experience，更新參數是從 replay buffer 中隨意挑一筆，降低了資料之間的相關性，讓 Neural Network 訓練避免偏差。
                                                • Target Network 避免了訓練目標經常地變動造成訓練效果差
                                                • 使用 Neural Network 替代 action value function 避免了 Q-Learning 的 table 維度過大訓練困難的問題

                                                # Double-DQN

                                                在 DQN 當中我們需要同時訓練兩個 model，也就是 θ\thetaθ 與 θ−\theta^-θ−。然而 DQN 的設定上對於目標被發現存在高估的問題，因此 Double-DQN 提出了解決這個高估問題的方法。

                                                原始 DQN 目標Double-DQN 目標

                                                高估的狀況如底下的綠線。紫線是目標函數，橘線是綠線與紫線的誤差，不難發現到確實都存在高估的狀況。

                                                然而使用了 Double-DQN 之後的誤差(藍線)就小到幾乎不存在了。

                                                Double-DQN 帶來的幾個好處

                                                • 讓 DQN 高估的問題消失，有更好的效果

                                                # Dueling DQN

                                                Dueling DQN 的概念仍然是透過 Neural Network 去學習 optimal action value function Q∗Q^*Q∗。 Action 的決定上採用了 ϵ\epsilonϵ-greedy。

                                                Image from Ziyu Wang et al. (2015)open in new window

                                                與 DQN 不同的地方在於他並不是直接去學習 Q∗Q^*Q∗，而是另外定義了一個 Advantage function AAA。

                                                A(s,a)=Q(s,a)−V(s) A(s, a) = Q(s, a) - V(s) A(s,a)=Q(s,a)−V(s)

                                                V(s)V(s)V(s) 就像是 baseline，表示著在當前這個 state sss 底下你預期可以拿到多好的 return，所以 A(s,a)A(s, a)A(s,a) 意義上就是在看每個 action 有多好多壞。

                                                透過 VVV 和 AAA 的總和就能夠得到 QQQ。上圖就是在最後分開成兩個輸出結果 VVV 和 AAA，最後合併成 QQQ。

                                                Q(s,a;θ)=V(s;θV)+A(s,a;θA) Q(s, a; \theta) = V(s; \theta_V) + A(s, a; \theta_A) Q(s,a;θ)=V(s;θV​)+A(s,a;θA​)

                                                Info

                                                在實務上為了避免像是 V(s)V(s)V(s) 都是 000，實際上跟 DQN 沒有差異的問題，因此細節上是還會對 A(s,a)A(s, a)A(s,a) 加上總和為 000 的限制。

                                                Q(s,a,θ,α,β)=V(a,θ,α)+[A(s,a,θ,β)−1∣A∣∑a′∈∣A∣A(s,a′,θ,β)] Q(s, a, \theta, \alpha, \beta) = V(a, \theta, \alpha) + \left[ A(s, a, \theta, \beta) - \frac{1}{|A|} \sum_{a' \in |A|}{A(s, a', \theta, \beta)} \right] Q(s,a,θ,α,β)=V(a,θ,α)+​A(s,a,θ,β)−∣A∣1​a′∈∣A∣∑​A(s,a′,θ,β)​

                                                α,β\alpha, \betaα,β 只是調整 VVV 和 AAA 兩部分影響程度的參數。

                                                把 Dueling DQN 搭配 Double DQN 之後可以得到底下的 Loss。

                                                L(θ)=E(s,a,r,y)∼D[(r+γQ(y,arg⁡max⁡b∈AQ(y,b;θ);θ−)−Q(s,a;θ))2] L(\theta) = \mathbb{E}_{(s, a, r, y) \sim D} \left[ \left( r + \gamma Q(y, \textcolor{red}{\arg \max_{b \in \mathcal{A}}{Q(y, b; \theta)}}; \theta^-) - Q(s, a; \theta) \right)^2 \right] L(θ)=E(s,a,r,y)∼D​[(r+γQ(y,argb∈Amax​Q(y,b;θ);θ−)−Q(s,a;θ))2]

                                                Dueling DQN 帶來了幾個好處

                                                • 使用 Advantage function 增加模型的更新與 exploration。

                                                # A3C

                                                在 DQN 當中使用了 Experience Reply 去避免訓練資料上的強關聯性，然而存在幾個缺點

                                                • 需要額外的 memory 去儲存 replay buffer
                                                • 需要 off-policy alogorithm，對於 online RL 來說可能導致收斂不穩定以及緩慢等問題

                                                A3C 的概念就如同火影忍者的影分身之術，讓每個分身在各自的環境當中訓練，訓練成效也就翻倍。

                                                Image from Arthur Juliani@Mediumopen in new window

                                                A3C 是使用 advantage actor-critic 的方式，會直接去學 policy 以及 value function。因此在參數上也就包含了兩項 θπ\theta_{\pi}θπ​ 以及 θv\theta_vθv​ 分別表示 policy 以及 value function 的參數。考慮在時間 ttt 往後看 kkk 步的更新，參照 A3C 的論文，兩個參數的 Loss 計算分別如下。

                                                • λ\lambdaλ 用來調整兩個 Loss 的影響力

                                                # Methodology

                                                # 基本想法

                                                Noisy-Net 的想法跟 Parameter Space Noise for Explorationopen in new window 的想法基本上是相同方向，都是要對 parameter space 去加上 noise。

                                                作法上，對於每個可訓練的參數拆解成 ζ=(μ,σ)\zeta = (\mu, \sigma)ζ=(μ,σ)，然後再透過 zero-mean 的 ϵ\epsilonϵ 增加 noise。也就是說對於一個參數 θ\thetaθ 我們會寫成：

                                                θ=defμ+σ⊙ϵ \theta \stackrel{\texttt{def}}{=} \mu + \sigma \odot \epsilon θ=defμ+σ⊙ϵ

                                                所以對於一個 Linear Layer 來說

                                                y=wx+b⇒y=(μw+σw⊙ϵw)x+(μb+σb⊙ϵb) y = wx + b \Rightarrow y = (\mu^w + \sigma^w \odot \epsilon^w)x + (\mu^b + \sigma^b \odot \epsilon^b) y=wx+b⇒y=(μw+σw⊙ϵw)x+(μb+σb⊙ϵb)

                                                就只是這樣而已，不要想太多！

                                                Tips

                                                刻意挑 zero-mean 的 noise 是為了採用底下的特性方便後續 Loss 的計算。

                                                Lˉ(ζ)=E[L(θ)] \bar{L}(\zeta) = \mathbb{E}[L(\theta)] Lˉ(ζ)=E[L(θ)]

                                                因此

                                                ∇Lˉ(ζ)=∇E[L(θ)]=E[∇μ,ΣL(μ+Σ⊙ϵ)] \nabla \bar{L}(\zeta) = \nabla \mathbb{E}[L(\theta)] = \mathbb{E} \left[\nabla_{\mu, \Sigma}{L(\mu + \Sigma \odot \epsilon)} \right] ∇Lˉ(ζ)=∇E[L(θ)]=E[∇μ,Σ​L(μ+Σ⊙ϵ)]

                                                Σ\SigmaΣ 包含了所有 σ\sigmaσ

                                                加上了 Monte-Carlo approximation 之後，可以用單一的 sample ξ\xiξ 去近似

                                                ∇Lˉ(ζ)≈∇μ,ΣL(μ+Σ⊙ξ) \nabla \bar{L}(\zeta) \approx \nabla_{\mu, \Sigma}L(\mu + \Sigma \odot \xi) ∇Lˉ(ζ)≈∇μ,Σ​L(μ+Σ⊙ξ)

                                                跟 OpenAI 提出的方法略為不同的地方在於他並不是直接對 network 的參數加上 Gaussian Noise，而是給了參數 ϵw,ϵb\epsilon^w, \epsilon^bϵw,ϵb 去決定要加怎樣的 noise。

                                                在每一個 episode 開始之前先把參數加上 noise，接下來這一整個 episode 就都是用這個 network 去訓練，意即在過程中不會對 noise 做調整。

                                                # 減少產 random number 時間

                                                這樣的做法下每一個 episode 都需要 random noise 在 weight 和 bias 上。假如 w∈Rq×p,b∈Rqw \in \mathbb{R}^{q \times p}, b \in \mathbb{R}^{q}w∈Rq×p,b∈Rq，那麼 ϵw∈Rq×p,ϵb∈Rq\epsilon^{w} \in \mathbb{R}^{q \times p}, \epsilon^{b} \in \mathbb{R}^{q}ϵw∈Rq×p,ϵb∈Rq，也就意味著需要 random 出 pq+qpq + qpq+q 個數值。

                                                上面基本的做法作者稱他為 Independent Gaussian noise，而接下來作者給出一個 Factorised Gaussian noise 的做法。

                                                基本上就是將 random number 拆分

                                                ϵi,jw=f(ϵi)f(ϵj)ϵjb=f(ϵj) \begin{align*} \epsilon_{i, j}^{w} &= f(\epsilon_i) f(\epsilon_j) \\ \epsilon_j^b &= f(\epsilon_j) \end{align*} ϵi,jw​ϵjb​​=f(ϵi​)f(ϵj​)=f(ϵj​)​

                                                其中 f(x)=sgn(x)∣x∣,ϵi∈Rq,ϵj∈Rpf(x) = \texttt{sgn}(x)\sqrt{|x|}, \epsilon_i \in \mathbb{R}^{q}, \epsilon_{j} \in \mathbb{R}^pf(x)=sgn(x)∣x∣​,ϵi​∈Rq,ϵj​∈Rp。

                                                如此一來，只需要產出 p+qp + qp+q 個 random number 也可以達到類似的效果。

                                                # DQN & Dueling DQN

                                                由於 DQN 和 Dueling DQN 是在 single-thread 上訓練，因此上述的 Random Overhead 會比較大，在這裡採用 Factorised Gaussian noise。

                                                現在 Network 的部分改用 Noisy Network，也就不需要再使用 ϵ\epsilonϵ-greedy 了。

                                                原本的 DQN 對 Loss 的定義如下。

                                                L(θ)=E(s,a,r,y)∼D[(r+γmax⁡b∈AQ(y,b;θ−)−Q(s,a;θ))2] L(\theta) = \mathbb{E}_{(s, a, r, y) \sim D} \left[ \left(r + \gamma \max_{b \in A} Q(y, b; \theta^-) - Q(s, a; \theta)\right)^2 \right] L(θ)=E(s,a,r,y)∼D​[(r+γb∈Amax​Q(y,b;θ−)−Q(s,a;θ))2]

                                                現在替換成 Noisy Net 的形式

                                                Lˉ(ζ)=E[E(s,a,r,y)∼D[r+γmax⁡b∈AQ(y,b,ϵ′;ζ−)−Q(s,a,ϵ;ζ)]2] \bar{L}(\textcolor{red}{\zeta}) = \textcolor{red}{\mathbb{E}} \left[ \mathbb{E}_{(s, a, r, y) \sim D} \left[ r + \gamma \max_{b \in A} Q(y, b, \textcolor{red}{\epsilon'}; \textcolor{red}{\zeta^-}) - Q(s, a, \textcolor{red}{\epsilon}; \textcolor{red}{\zeta}) \right]^2 \right] Lˉ(ζ)=E[E(s,a,r,y)∼D​[r+γb∈Amax​Q(y,b,ϵ′;ζ−)−Q(s,a,ϵ;ζ)]2]

                                                最外層的期望值是對 ϵ\epsilonϵ 和 ϵ′\epsilon'ϵ′

                                                同樣地也可以對 Dueling DQN 做修改。原本的定義為

                                                L(θ)=E(s,a,r,y)∼D[(r+γQ(y,arg⁡max⁡b∈AQ(y,b;θ);θ−)−Q(s,a;θ))2] L(\theta) = \mathbb{E}_{(s, a, r, y) \sim D} \left[ \left( r + \gamma Q(y, \arg \max_{b \in \mathcal{A}}{Q(y, b; \theta)}; \theta^-) - Q(s, a; \theta) \right)^2 \right] L(θ)=E(s,a,r,y)∼D​[(r+γQ(y,argb∈Amax​Q(y,b;θ);θ−)−Q(s,a;θ))2]

                                                現在替換成 Noisy Net 的形式

                                                Lˉ(ζ)=E[E(s,a,r,y)∼D[r+γQ(y,arg⁡max⁡b∈AQ(y,b,ϵ′′;ζ),ϵ′;ζ−)−Q(s,a,ϵ;ζ)]2] \bar{L}{(\textcolor{red}{\zeta})} = \textcolor{red}{\mathbb{E}}\left[\mathbb{E}_{(s, a, r, y) \sim D} \left[ r + \gamma Q(y, \arg \max_{b \in \mathcal{A}}{Q(y, b, \textcolor{red}{\epsilon''}; \textcolor{red}{\zeta})}, \textcolor{red}{\epsilon'}; \textcolor{red}{\zeta^-}) - Q(s, a, \textcolor{red}{\epsilon}; \textcolor{red}{\zeta}) \right]^2 \right] Lˉ(ζ)=E[E(s,a,r,y)∼D​[r+γQ(y,argb∈Amax​Q(y,b,ϵ′′;ζ),ϵ′;ζ−)−Q(s,a,ϵ;ζ)]2]

                                                # Distributed A3C

                                                由於 A3C 是在 multi-thread 上訓練，因此不太需要考慮上述的 Random Overhead，在這裡採用 Independent Gaussian noise 即可。

                                                現在 Network 的部分改用 Noisy Network，也就不需要再使用 Entropy function 了。

                                                # Analysis

                                                為了進一步去釐清這樣的做法為什麼是可行、合理的，作者進一步去研究。

                                                回顧一下我們加上 Noise 的方法，是把一個可訓練參數拆成 ζ=(μ,σ)\zeta = (\mu, \sigma)ζ=(μ,σ)，再額外多一個 Noise ϵ\epsilonϵ。

                                                θ=defμ+σ⊙ϵ \theta \stackrel{\texttt{def}}{=} \mu + \sigma \odot \epsilon θ=defμ+σ⊙ϵ

                                                理想上，我們最後的 Loss 應該要能夠好好收斂，也就是說最後的 solution 應該要是 deterministic。那麼這裡加上的 ϵ\epsilonϵ 就應該隨著訓練慢慢被忽視，作用只在於訓練的前中期提供 exploration。因此，我們也就會期待 σ\sigmaσ 這個參數會漸漸趨近於 000 了！

                                                定義底下的平均

                                                Σˉ=1Nweights∑i∣σiw∣ \bar{\Sigma} = \frac{1}{N_{\texttt{weights}}}\sum_{i}{|\sigma_i^w|} Σˉ=Nweights​1​i∑​∣σiw​∣

                                                作者發現在每一個遊戲當中最後一個 Layer 的 Σˉ\bar{\Sigma}Σˉ 都是會逐漸趨近於 000 的，然而若觀察倒數第二個 Layer 卻並不一定了，有些甚至是遞增的。也就是說，其實 NoisyNet 並不會都得出 deterministic solution。

                                                此外，透過 Σˉ\bar{\Sigma}Σˉ 的曲線也可以發現到在不同的遊戲當中他們的更新曲線相當地不同，也說明了實際上這樣的更新方式是能夠依照不同的遊戲去適應的。
